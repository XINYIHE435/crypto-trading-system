## 问题1: 回测未产生任何交易

### 问题分析
通过添加调试模式发现，回测没有交易的根本原因是**参数阈值设置过高**：

#### 实际市场数据
- **价差范围**: 0.003% - 0.01%
- **资金费率差**: 0.01% - 0.03%

#### 原始配置（导致0交易）
- **X (差价触发阈值)**: 0.5% ← 比实际价差大100倍
- **Y (费率差触发阈值)**: 0.1% ← 比实际费率差大10倍

### 解决方案

#### 1. 添加调试功能
在 `src/arbitrage_system.py` 中添加了详细的调试输出：
- 每100行或价差接近阈值时打印详细信息
- 显示实际价格、价差、费率差
- 显示各项开仓条件的检查结果
- 显示为什么没有触发交易

**使用方法：**
```bash
python main.py backtest --debug --symbols BTCUSDT
```

#### 2. 打印实际参数
在回测开始时自动打印所有配置参数：
```
📊 回测参数配置:
  差价触发阈值 X = 0.5%
  费率差触发阈值 Y = 0.1%
  可忽视差价 A = 0.01%
  可忽视费率差 B = 0.005%
  历史窗口 N = 4 小时
  费率不利持续 M = 2 小时
  盈利目标 P = 1.0%
  止损阈值 Q = 1.0%
  最大持仓数 = 5
  交易手续费 = 0.1%
```

#### 3. 推荐的参数配置

**激进策略（高频交易）：**
```bash
python main.py backtest --X 0.01 --Y 0.01
```
- 差价阈值: 0.01%
- 费率差阈值: 0.01%
- 结果: 1213笔交易/5个币对

**保守策略（低频交易）：**
```bash
python main.py backtest --X 0.05 --Y 0.05
```
- 差价阈值: 0.05%
- 费率差阈值: 0.05%

**中性策略：**
```bash
python main.py backtest --X 0.02 --Y 0.02
```

## 问题2: 可视化图表未更新

### 问题分析
`data/results/` 文件夹中的图表文件日期为 5月15日，没有自动更新。

### 解决方案

#### 1. 添加自动图表生成功能
在 `src/data/run_arbitrage_backtest.py` 中添加了 `generate_visualizations()` 方法：

**功能特性：**
- 为每个交易对生成4个子图的时间序列分析
- 自动保存到 `data/results/` 目录
- 图表包含：
  1. **交易所价格对比** - Binance vs KuCoin/Bybit
  2. **价差百分比** - 显示触发阈值线
  3. **资金费率对比** - 两个交易所的费率走势
  4. **资金费率差** - 显示触发阈值线

#### 2. 集成到回测流程
修改 `main.py`，在每次回测完成后自动生成图表：

```python
report = runner.run_all_symbols()
runner.print_summary(report)
runner.save_results(report)

# 自动生成可视化图表
runner.generate_visualizations(report)
```

#### 3. 生成的图表

运行回测后，以下图表自动更新：

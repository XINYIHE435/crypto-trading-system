## 2. 马丁双向网格策略

### 2.1 策略原理

双向马丁格尔网格策略同时在多头和空头两个方向建立网格，通过价格波动捕捉收益。

**核心逻辑**：
```
价格上涨超过网格线 → 开多单（追涨）
价格下跌超过网格线 → 开空单（追跌）
```

### 2.2 类结构

```python
# 文件: Martingale/main.py
```

### 2.4 开仓逻辑

```python
def on_tick(self, current_price: float, price_data: Optional[Dict] = None):
    """主循环: 处理实时价格"""
    
    # 1. 检查止盈止损
    if self.check_take_profit(current_price):
        return

    # 2. 计算网格间距
    grid_spacing = self.get_grid_spacing(current_price)
    
    # 3. 检查网格触发条件
    price_diff = current_price - self.baseline_price
    
    # 上涨区间：价格 > 基准价 → 开多单
    if price_diff > 0:
        grid_index = int(price_diff / grid_spacing)
        if grid_index > self.long_state.current_level:
            self.open_order('long', current_price)

    # 下跌区间：价格 < 基准价 → 开空单
    elif price_diff < 0:
        grid_index = int(abs(price_diff) / grid_spacing)
        if grid_index > self.short_state.current_level:
            self.open_order('short', current_price)
```

### 2.5 马丁倍投机制

```python
def get_lot_size(self, direction: str) -> float:
    """计算下单手数"""
    state = self.long_state if direction == 'long' else self.short_state
    level = state.current_level
    
    # 普通层: 固定手数
    if level < self.martingale_start_level:
        return self.base_size
    else:
        # 马丁层: 上一单手数 × 倍数
        if state.positions:
            return round(state.positions[-1].size * self.multiplier, 6)
        return self.base_size
```

- 第2层: 0.016

### 2.7 网格间距计算

```python
def get_grid_spacing(self, current_price: float) -> float:
    """获取当前网格间距"""
    
    if self.grid_spacing_type == GridSpacingType.FIXED:
        # 固定点数
        return self.grid_step
    
    elif self.grid_spacing_type == GridSpacingType.PERCENTAGE:
        # 百分比: 间距 = 当前价格 × 百分比
        return current_price * (self.grid_percentage / 100.0)
    
    elif self.grid_spacing_type == GridSpacingType.ATR:
        # ATR动态: 间距 = ATR × 倍数
        atr = self.calculate_atr()
        return atr * self.atr_multiplier
```

### 2.8 流程图

```
启动策略 → 设定基准价
    ↓
收到新价格 (on_tick)
    ↓
检查止盈/止损 → 触发 → 平仓 → 重设基准价
    ↓ 未触发
计算网格间距
    ↓
价格上涨超过网格线？→ 是 → 开多单
    ↓ 否
价格下跌超过网格线？→ 是 → 开空单
    ↓
等待下一个价格
```

## 3. 跨交易所套利策略

### 3.1 策略原理

在两个交易所（A=Binance, B=KuCoin）之间利用价差和资金费率差获利。

**核心逻辑**：
```
当 A价格 > B价格 → 做空A + 做多B → 等待价差收敛 → 平仓获利
当 A价格 < B价格 → 做多A + 做空B → 等待价差收敛 → 平仓获利
```

### 3.2 类结构

```python
# 文件: src/arbitrage_system.py
```

### 3.4 方向判断

```python
def check_direction(self, price_direction: int, funding_direction: int) -> Direction:
    """
    检查方向是否一致
    
    价差方向: 1 表示 A > B, -1 表示 A < B
    费率方向: 1 表示 A > B（做空A可收取费率）
    """
    if price_direction == funding_direction:
        return Direction.SAME      # 相同方向
    return Direction.DIFFERENT     # 不同方向
```

### 3.5 开仓条件 (6种)

#### 差价套利

| 条件 | 触发条件 |
|------|----------|
| **条件a (相同方向)** | 价差≥X + 价差>历史平均 + 相同方向 + 费率差<Y |
| **条件b (不同方向)** | 价差≥X + 价差>历史平均 + 不同方向 + 费率差<B |

```python
def check_price_spread_arbitrage_condition_a(self, ...):
    """差价套利 - 条件a（相同方向）"""
    return (
        price_spread_pct >= self.config.X and
        price_spread_pct > historical_avg and
        direction == Direction.SAME and
        funding_spread_pct < self.config.Y
    )
```

#### 资金费率套利

| 条件 | 触发条件 |
|------|----------|
| **条件a (相同方向)** | 费率差≥Y + 历史N小时都≥Y + 相同方向 + 价差<X |
| **条件b (不同方向)** | 费率差≥Y + 历史N小时都≥Y + 不同方向 + 价差<A |

#### 组合套利

| 条件 | 触发条件 |
|------|----------|
| **组合 (仅相同方向)** | 价差≥X + 价差>历史平均 + 费率差≥Y + 历史都≥Y + 相同方向 |

**特殊规则**：价差≥X 且 费率差≥Y 但方向不同 → **不开仓**

### 3.6 平仓条件 (按套利类型)

#### 差价套利场景一 (条件a开仓)
```
a. 价格回归盈利 → 立即平仓
b. 费率反转止损 → 价差无利 + 费率反转 + 费率>A + 持续>M小时
c. 价差亏损止损 → 价差亏损≥Q
```

#### 差价套利场景二 (条件b开仓)
```
a. 价格回归盈利 → 立即平仓
b. 费率扩大止损 → 价差无利 + 需支付费率 + 费率从<A变>B + 持续>M小时
c. 价差亏损止损 → 价差亏损≥Q
```

#### 资金费率套利
```
a. 费率收敛/反转 → 费率差<B 或 从赚变付 → 立即平仓
b. 价差盈利平仓 → 费率仍可赚 + 价差盈利≥P
c. 价差亏损止损 → 价差亏损≥Q
```

#### 组合套利
```
a. 费率收敛/反转/价差盈利 → 三者任一触发 → 立即平仓
b. 价差亏损止损 → 价差亏损≥Q
```

### 3.7 盈亏计算

```python
def close_position(self, position, price_a, price_b, ...):
    """平仓并计算盈亏"""
    
    # 根据套利方向计算价差盈亏
    if position.short_a_long_b:
        # 做空A做多B
        pnl_a = (position.open_price_a - price_a) * size  # A空头盈亏
        pnl_b = (price_b - position.open_price_b) * size  # B多头盈亏
    else:
        # 做多A做空B
        pnl_a = (price_a - position.open_price_a) * size  # A多头盈亏
        pnl_b = (position.open_price_b - price_b) * size  # B空头盈亏
    
    price_spread_pnl = pnl_a + pnl_b
    
    # 总盈亏 = 价差盈亏 + 累计资金费率收益 - 手续费
    total_pnl = price_spread_pnl + position.accumulated_funding_pnl - total_fee
```

### 3.8 资金费率结算

```python
def settle_funding_rate(self, position, funding_a, funding_b, current_time):
    """每8小时结算一次资金费率"""
    
    if hours_since_settlement >= 8.0:
        position_value = position.position_size * position.open_price_a
        
        if position.short_a_long_b:
            # 做空A做多B：收取A的费率，支付B的费率
            funding_income = (funding_a - funding_b) * position_value
        else:
            # 做多A做空B：收取B的费率，支付A的费率
            funding_income = (funding_b - funding_a) * position_value
        
        position.accumulated_funding_pnl += funding_income
```

### 3.9 K线间隔适配

```python
@dataclass
class ArbitrageConfig:
    kline_interval: str = '30T'
    kline_interval_minutes: int = 30
    
    def get_periods_for_hours(self, hours: int) -> int:
        """计算N小时对应的K线根数"""
        hours_in_minutes = hours * 60
```

## 4. 数据聚合模块

### 4.1 模块概述

将高频K线数据（如30分钟）聚合为低频数据（如1小时、4小时等）。

**文件位置**: `src/data/data_aggregator.py`

### 4.2 核心函数

```python
def aggregate_data(
    df: pd.DataFrame,
    interval: str,
    agg_rules: Dict[str, Any],
    timestamp_col: str = None,
    dropna: bool = True
) -> pd.DataFrame:
    """
    将高频数据聚合为低频数据的通用函数
    
    参数:
        df: 源数据 DataFrame
        interval: 时间间隔 ('30T', '1H', '4H', '1D' 等)
        agg_rules: 聚合规则字典
        timestamp_col: 时间戳列名
        dropna: 是否删除NaN行
    
    返回:
        聚合后的 DataFrame
    """
    # 标准化时间间隔
    normalized_interval = normalize_interval(interval)
    
    # 执行重采样和聚合
    df_aggregated = df.resample(normalized_interval).agg(agg_rules)
    
    return df_aggregated
```

### 4.4 两种模式对比

| 模式 | 价格聚合方式 | 适用场景 |
|------|-------------|---------|
| **OHLC** | open=first, high=max, low=min, close=last | 标准K线分析、技术指标计算 |
| **MEAN** | 全部取mean | 趋势平滑分析、降噪 |

| 参数 | 范围 | 推荐值 | 说明 |
|------|------|--------|------|
| base_size | 0.001~1.0 | 0.01 | 根据资金量调整 |
| multiplier | 1.0~3.0 | 1.5~1.8 | 倍数越大风险越高 |
| max_levels | 3~15 | 6~10 | 建议不超过10 |
| grid_percentage | 0.1~2.0 | 0.3~0.8 | 波动大用大值 |
| max_floating_loss | 50~500 | 100~200 | 不超过本金10% |

#### 套利策略

| 参数 | 范围 | 推荐值 | 说明 |
|------|------|--------|------|
| X | 0.01~1.0 | 0.03~0.1 | 价差越小交易越频繁 |
| Y | 0.001~0.1 | 0.002~0.01 | 费率差触发条件 |
| P | 0.01~0.5 | 0.02~0.05 | 止盈目标 |
| Q | 0.01~0.5 | 0.03~0.1 | 止损阈值 |
| N | 4~24 | 8 | 历史数据小时数 |
| M | 2~12 | 4 | 不利条件持续时间 |

### B. 常见问题

**Q1: K线间隔改变后，策略参数需要调整吗？**

A: 
- 马丁策略：`atr_period` 按K线根数计算，间隔变化后时间跨度会变
- 套利策略：参数 N、M 是按小时计算的，系统会自动转换为K线根数

**Q2: OHLC模式和MEAN模式该选哪个？**

A:
- OHLC：适合技术分析、标准K线形态识别
- MEAN：适合趋势分析、降噪平滑

**Q3: 如何添加新的聚合规则？**

A: 在 `data_aggregator.py` 中定义新的规则字典：

```python
MY_RULES = {
    'close': 'last',
    'volume': 'sum',
    'custom_col': lambda x: x.quantile(0.75)  # 自定义函数
}
```

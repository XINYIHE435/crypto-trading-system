# 套利系统快速使用指南

## 🚀 快速开始

### 1. 标准回测（推荐）
```bash
python main.py backtest --X 0.01 --Y 0.01
```
这将：
- ✅ 运行所有5个交易对的回测
- ✅ 生成详细的回测报告
- ✅ 自动生成可视化图表到 `data/results/`

### 2. 调试模式（排查问题）
```bash
python main.py backtest --debug --symbols BTCUSDT
```
这将：
- 🔍 只测试 BTCUSDT
- 🔍 显示每100行的详细信息
- 🔍 打印为什么没有触发交易
- 🔍 显示所有开仓条件的检查结果

### 3. 下载最新数据
```bash
python main.py download --days 30
```

## 📊 参数说明

### 关键参数
- `--X`: 差价触发阈值（%），默认 0.5%
- `--Y`: 资金费率差触发阈值（%），默认 0.1%
- `--symbols`: 交易对列表，默认全部5个
- `--debug`: 启用调试模式

### 推荐配置

#### 🔥 激进策略（高频）
```bash
python main.py backtest --X 0.01 --Y 0.01
```
- 预期交易次数: ~1200笔/30天
- 适合: 测试策略逻辑

#### 🛡️ 保守策略（低频）
```bash
python main.py backtest --X 0.05 --Y 0.05
```
- 预期交易次数: ~100笔/30天
- 适合: 降低交易成本

#### ⚖️ 中性策略（推荐）
```bash
python main.py backtest --X 0.02 --Y 0.02
```
- 平衡交易频率和成本

## 🔧 常见问题

### Q1: 为什么没有交易？
**A:** 检查参数阈值是否过高。使用调试模式查看实际价差：
```bash
python main.py backtest --debug --symbols BTCUSDT
```

实际市场数据通常：
- 价差: 0.003% - 0.01%
- 费率差: 0.01% - 0.03%

如果 X=0.5%, Y=0.1%，阈值太高，不会触发交易。

### Q2: 为什么策略亏损？
**A:** 可能的原因：
1. **手续费过高**: 当前 0.1%，每笔交易成本约 0.4%
2. **止盈止损参数**: P=1.0%, Q=1.0% 可能不合理
3. **交易频率过高**: 降低阈值会增加交易次数和手续费

### Q3: 如何优化参数？
**A:** 使用参数优化功能：
```bash
python main.py backtest --optimize
```

或手动测试不同参数组合。

### Q4: 图表没有更新？
**A:** 图表会在每次回测后自动生成。确保：
1. 运行的是标准回测（非调试模式）
2. 检查 `data/results/` 目录的文件时间戳
